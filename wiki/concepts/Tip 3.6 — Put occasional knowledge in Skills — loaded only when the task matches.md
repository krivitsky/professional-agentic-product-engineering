# Tip 3.6 — Put occasional knowledge in Skills — loaded only when the task matches
**Part of:** [[Tier 3 — Context Management]] · tip 6 of 8
**Source:** `guide.md #tip-3-6` (root — canonical, not copied)
**Created:** 2026-07-04

> **Instead of:** cramming the migration protocol into CLAUDE.md.
>
> **Prefer:** a Skill that auto-loads only when the task matches its description.

A Skill is a folder with a `SKILL.md` that loads on-demand by its description — so occasional knowledge stays out of the always-on context budget until it's needed.
